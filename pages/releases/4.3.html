Outstanding features

Interactive music support

Like music to your ears

Always wanted to create a dynamic music system that would react to the context of your game?

Now you can!

With the added resources AudioStreamInteractive, AudioStreamPlaylist, and AudioStreamSynchronized, you can create complex, layered interactive music and transitions.

2D physics interpolation

More frames, more!

Physics interpolation is the key to decouple physics ticks and display frame rates. What it does is to create dynamically frames inbetween the last physics position and the current one.

While it's not a silver bullet, it can greatly improve support for high-refresh displays. Or, it can enable you to display smooth animations with a low physics tick rate.

Visual shader editor overhaul

Throw shades at ease

The visual shader editor has been revamped from the ground up!

Selected nodes are now highlighted! You can also now add reroute nodes to space out content and you can insert frames in order to organize neatly your shader.

New TileMapLayer node

Layer up with this new node

Now gone the era of handling poorly layers of TileMap.

With Godot 4.3, we deprecated the TileMap node in order and we created the TileMapLayer node. By doing this, we reduced clutter in the inspector, we simplified the API, and layers are now in line with common Godot design patterns.

Single-threaded Web export

Export easily for the Web

We orignially aimed for the stars, but browsers are still down to Earth.

By bringing back single-threaded export for the Web platform, we got more compatible across browsers and we eliminated the need of complicated server-side configurations.

2D

New Parallax2D node

Just add water!

The new Parallax2D node aims to greatly simplify the use of parallax in your games. You don't have to deal with CanvasLayer anymore.

We even included a nifty converter!

Pixel art stability

Stabilize those pixels

You activated "Snap 2D Transforms to Pixel" in the project settings and your pixel art still giggles? Your character doesn't snap to the ground?

In 4.3, new algorithms are used to stabilize pixel positions which should fix these issues.

Animation

New SkeletonModifier3D abstract node

This new abstract node was made to animate Skeleton3D outside of AnimationMixer and is now the base class for several existing nodes.

You can't use it directly, but you can create your own custom SkeletonModifier3D node easily using a simple script.

You'll be able to implement the new _process_modification() function to modify the bones how ever you like!

Skeletal animation import options

Retarget like a pro

We implemented new import options in order to assist you in retargeting animation sets, especially those using the .fbx format.

From importing a rest pose as RESET animation to a retargeting option to use a template for silhouette, we've made sure that you'll be able to fully use all the skeletons in your closet.

More keyframe manipulations

Manipulating keyframes is now easier

You can finally select, copy, paste, and duplicate keyframes.

What a relief!

C#

Display

Direct3D 12 rendering driver

We tamed the green machine

This new driver enables new Microsoft platforms to be supported, such as Microsoft Store (UWP) and Windows on ARM.

It even enables custom ports to a certain green platform!

Wayland support for Linux/*BSD

Following the Way

Wayland is a newer, more modern display server that can be used on Linux and some BSD-derived OSes.

Builtin Wayland support give us access to a much cleaner API as well as more advanced features for window management.

Editor

FBX native support

FBX your game

Godot now handles .fbx files natively, so you can import 3D models and animations from an industry-standard file format more easily.

FileSystem dock can be moved to the bottom section of the editor

Want more space to see your files? You can now move the FileSystem dock to the bottom section of the editor.

Especially practical if you're used to find your files there.

GDScript

Binary tokenization on export

Lean and mean exported GDScript

With this new export option, you can reduce the size of the exported GDScript code. But it also offers a way to obfuscate your source code.

While it doesn't protect your work from being decompiled, it at least hides your source code from being available publicly.

Built-in methods and utility functions usable as Callable

You have an array and you want to print each element without creating a for loop?

By making built-in methods and utility functions usable as Callable makes it possible to call variadic functions such as print() with Callable.callv() easily, without the need of boilerplate code.

You can now call this:
print.callv(my_array)

@export_storage property hint

Addon developers, you can now use @export_storage to store hidden values in a scene.

Useful if you need to store custom data that isn't meant to be edited in the inspector.

@export_custom property hint

You need specific types of export? We got you covered.

You can now use @export_custom to export a value suited to your needs.

GDScript autocompletion fixes

GDScript circular dependencies between scenes and scripts

A lot of issues with circular dependencies between scenes and scripts have been fixed!

Not only this makes you less worry about cyclic dependencies, but it makes scenes more resilient and stable.

Web

Web Audio samples

Enhanced audio, rock on!

With great power comes great responsibility when it comes to the Web.

As there was audio issues when running games single-threaded, we updated the engine to support audio samples. This enabled us to leverage the Web Audio API in order to deliver seamlessly low-latency and high-quality audio.

Splash screen is now showing up

Not only the Web progress bar has been reworked and optimized, but the splash screen is now showing up before the game loads!

Don't worry: if no boot splash screen has been set, the classic Godot boot splash will be seen, as it is currently displayed on other platforms.

XR

Composition layers to allow 2D viewports to be displayed at much higher quality

Fixed tracking issues in the native mobile interface

Standardized and enhanced support for hand/finger tracking, body tracking, and face tracking

Support for various Meta extensions such as hand models, render models, and scene discovery

Improvements to foveated rendering/vrs, passthrough, and various other supporting systems
