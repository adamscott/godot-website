Outstanding features

Interactive music support

Like music to your ears

Always wanted to create a dynamic music system that would react to the context of your game?

Now you can!

With the added resources AudioStreamInteractive, AudioStreamPlaylist, and AudioStreamSynchronized, you can create complex, layered interactive music and transitions.

Read more

2D physics interpolation

More frames, more!

Physics interpolation is the key to decouple physics ticks and display frame rates. What it does is to create dynamically frames inbetween the last physics position and the current one.

While it's not a silver bullet, it can greatly improve support for high-refresh displays. Or, it can enable you to display smooth animations with a low physics tick rate.

Read more

Visual shader editor overhaul

Throw shades at ease

The visual shader editor has been revamped from the ground up!

Selected nodes are now highlighted! You can also now add reroute nodes to space out content and you can insert frames in order to organize neatly your shader.

Read more

Single-threaded Web export

Export easily for the Web

We orignially aimed for the stars, but browsers are still down to Earth.

By bringing back single-threaded export for the Web platform, we got more compatible across browsers and we eliminated the need of complicated server-side configurations.

Read more

New TileMapLayer node

Layer up with this new node

Now gone the era of handling poorly layers of TileMap.

With Godot 4.3, we deprecated the TileMap node in order and we created the TileMapLayer node. By doing this, we reduced clutter in the inspector, we simplified the API, and layers are now in line with common Godot design patterns.

Read more

New SkeletonModifier3D abstract node

This new abstract node was made to animate Skeleton3D outside of AnimationMixer and is now the base class for several existing nodes.

You can't use it directly, but you can create your own custom SkeletonModifier3D node easily using a simple script.

You'll be able to implement the new _process_modification() function to modify the bones how ever you like!

From importing a rest pose as RESET animation to a retargeting option to use a template for silhouette, we've made sure that you'll be able to fully use all the skeletons in your closet.

More robust reload process

Issues have been fixed.

One of the most annoying issues that C# users have with Godot have been fixed.

When a C# project is re-built, scripts need to be reloaded. Unfortunately, a lot of things can go wrong during this process. It usually left users in an invalid state with no other option but to re-open the Godot editor entirely.

Now, the process has been improved and users will be able to appreciate the new robustness of the reloads.

Read more

Better support of generic C# classes

Patience yields results!

This feature has been requested for a long time, and it's finally here.

While the implementation has some limitations, you'll be finally able to use (more comfortably) generic C# classes in your code.

Read more

Preventing export without C# code

The .solution and .csproj files are required for Godot to build C# projects. When they were missing, the export was successful but it was missing the C# code. This meant that you'd end up running into runtime errors.

From now on, the exporter will inform you about that fact and stop exporting.

Built-in methods and utility functions usable as Callable

You have an array and you want to print each element without creating a for loop?

By making built-in methods and utility functions usable as Callable makes it possible to call variadic functions such as print() with Callable.callv() easily, without the need of boilerplate code.

You can now call this:

Addon developers, you can now use @export_storage to store hidden values in a scene.

Useful if you need to store data on a node or a resource that need to be accessed by the addon, but that isn't meant to be edited by the user in the inspector.

@export_custom property hint

You need specific types of export? We got you covered.

You can now use @export_custom to export a value suited to your needs.

It allows you to set a custom hint, hint string, and usage flags for the exported property. Great for highly specific exports and for addon developers.

Navigation

Navigation mesh chunks baking

Split and bake!

If you're working on a project with large game worlds, this update will definitely be interesting to you.

This new feature allows you to split a large area into source geometry chunks and bake chunk navigation meshes. You'll be then able to load those chunks on-demand, saving up computing resources.

These chunks will have border edges aligned so they can be merged efficiently by the navigation map.

Read more

Navigation obstacle option to affect baking

Our new cookie cutter

NavigationObstacle2D and NavigationObstacle3D now have two new options to affect navigation mesh baking.

With these options, you can either affect the generated navigation mesh, or carve into it directly, as if you were using a stencil.

This is especially useful for 3D navigation mesh baking as it can help discarding internal navigation meshes that appears in geometry.

Read more

Navigation path simplification

With the new "Simplify Path" option on navigation agent nodes, it is now possible to remove non-critical path points.

This is useful to resolve some issues that comes with agent movement in large areas. It also helps agents that use navigation points to steer as it leaves only the critical path points as targets.

Read more

Rendering

Premultiplied alpha blending in 3D shaders

Premultiplied alpha is now a mixing mode in 3D shaders. This enables you to create better looking flames and fireworks, as the same particle now can have the properties of additive (for the flame) and mixing (for the smoke) blending.

See on the video how the left barrel flames which is using additive blending don't end up in smoke like the other barrel flames which use premultiplied alpha blending.

Read more
